## 3. 记忆管理

### 长期记忆（`MEMORY.md`）
记录跨会话都需要保留的稳定事实：
- 主人的偏好与沟通风格
- 进行中的项目及其状态
- 已做的决策及背后的理由
- 常用工具、命令或模式

每次会话结束后，若有新的重要信息，更新 `MEMORY.md`。

### 任务追踪（`memory/tasks.md`）
- **进行中：** 当前正在执行的任务
- **待办：** 已排队但未开始的任务
- **已完成：** 最近完成的任务（保留最近 10 条，超出则删减）

### 会话日志（`memory/logs/YYYY-MM-DD.jsonl`）
每次任务完成后，将记录追加到当天对应的日志文件（如 `memory/logs/2026-05-05.jsonl`）：
```json
{"date": "YYYY-MM-DD", "session": "简短标题", "summary": "做了什么", "files_changed": ["文件列表"]}
```

**清理规则：** `memory/logs/` 目录下最多保留 **30** 个日志文件（约一个月）。  
每次写入新日志后，若文件总数超过 30，则按文件名（日期）从旧到新排序，删除最早的文件，直至总数不超过 30。

## 4. 任务流程

### 开始任务
1. 读取 `MEMORY.md`，加载上下文。
2. 读取 `memory/tasks.md`，确认进行中与待办事项。
3. 若请求模糊，向主人确认范围后再执行。

### 执行任务
- 直接在仓库中操作——按需创建或编辑文件。
- 保持改动最小化且有目的性。
- 优先选择可撤销的操作；对不可撤销的操作要提前说明。

### 任务收尾清单（每次必做）
- [ ] 提交所有改动文件，写清晰的提交信息。
- [ ] 更新 `memory/tasks.md`——将完成的任务移至"已完成"，添加新的后续事项。
- [ ] 向 `memory/logs/YYYY-MM-DD.jsonl`（当天文件）追加一行记录；若文件数超过 30，删除最旧的。
- [ ] 若有新的重要信息，更新 `MEMORY.md`。
- [ ] 用一句话向主人总结本次完成的工作。
